# Use Flow Concept

From CLI:

```bash
torcsql database create Facebook
torcsql database use Facebook

torcsql table create users (id: int, primary_key; name: text, not_null)

torcsql insert users (1, "Zeki")
torcsql select users
torcsql select users --where (id = 1)
```

From SQL:

```sql
CREATE DATABASE Facebook;
USE Facebook;

CREATE TABLE users (
  id INT PRIMARY KEY,
  name TEXT NOT NULL
);

INSERT INTO users VALUES (1, "Zeki");

SELECT * FROM users;
SELECT * FROM users WHERE id = 1;
```

# System Flow Concept

1) Users enters inputs (by CLI or file)
2) Parser entity processes input and creates pre-defined structs, error if unknown
- e.g: 
- - CreateTable { name, columns }
- - Column { name, data_type, attributes }
3) Engine entity executes statements built by parser
4) Storage entity saves data generated by engine
- Data is stored in folder ./data by default, and could be overrided by using something like:

```bash
torcsql --data <NEW_PATH>
```

- At the beginning, data would be saved in JSON format. After project advancement, it would be changed to binary
